# Assumptions and Constraints: PRCI v2.0

## User Assumptions

### 1. Technical Proficiency
- **Basic Computer Skills**: Users possess fundamental computer operation skills
- **Internet Access**: Reliable internet connectivity is available for system access
- **Device Compatibility**: Users have access to compatible devices (Windows PC or modern web browser)
- **Software Updates**: Users maintain updated browsers and necessary software components

### 2. Academic Context
- **Student Status**: Primary users are enrolled university students
- **Academic Workload**: Users have regular academic assignments and deadlines
- **Digital Tool Usage**: Students utilize digital tools for academic activities
- **Institutional Support**: Educational institutions provide basic technical support

### 3. Behavioral Patterns
- **Consistent Patterns**: Procrastination behaviors exhibit identifiable patterns
- **Digital Footprints**: Academic activities leave traceable digital patterns
- **Self-Awareness**: Users have some awareness of their procrastination tendencies
- **Willingness to Improve**: Users are motivated to address procrastination issues

### 4. Data Availability
- **Sufficient Data**: Users generate adequate behavioral data for pattern analysis
- **Data Quality**: Collected data is accurate and representative of user behavior
- **Historical Patterns**: Sufficient historical data exists for predictive modeling
- **Data Continuity**: Users maintain consistent system usage over time

## Academic Time Constraints

### 1. Semester Duration
- **Development Timeline**: Complete system development within single semester (16 weeks)
- **Phase Distribution**: Strict adherence to phase-wise development schedule
- **Milestone Deadlines**: Fixed deadlines for each development phase
- **Final Submission**: System completion and documentation by semester end

### 2. Resource Limitations
- **Development Hours**: Limited development time due to concurrent academic courses
- **Team Availability**: Coordination constraints among team members' schedules
- **Testing Period**: Limited time for comprehensive testing and validation
- **Documentation Time**: Constraints on thorough documentation preparation

### 3. Academic Calendar Constraints
- **Exam Periods**: Reduced development capacity during examination weeks
- **Holiday Breaks**: Limited development during academic holidays
- **Presentation Schedule**: Fixed dates for project presentations and evaluations
- **Viva Voce Preparation**: Time allocation for final defense preparation

### 4. Institutional Requirements
- **Evaluation Criteria**: Compliance with university project evaluation standards
- **Ethics Approval**: Time required for institutional ethics review and approval
- **Advisor Availability**: Limited access to project supervisor for guidance
- **Resource Access**: Constraints on university computing resources and facilities

## Hardware Constraints

### 1. Development Environment
- **Personal Computers**: Development limited to personal laptop/desktop computers
- **Processing Power**: Limited CPU and GPU capabilities for model training
- **Memory Limitations**: RAM constraints affecting data processing and model operations
- **Storage Capacity**: Limited disk space for data storage and model files

### 2. Testing Infrastructure
- **Single Machine Testing**: No dedicated testing servers or cloud infrastructure
- **Load Testing Limitations**: Inability to conduct large-scale performance testing
- **Network Simulation**: Limited capability to test various network conditions
- **Device Diversity**: Restricted testing across different device configurations

### 3. Deployment Constraints
- **Local Hosting**: System deployment limited to local machine or basic web hosting
- **Server Limitations**: No access to enterprise-grade server infrastructure
- **Bandwidth Restrictions**: Limited network bandwidth for user access
- **Backup Systems**: Limited automated backup and recovery capabilities

## Software Constraints

### 1. Technology Stack Limitations
- **Open Source Preference**: Preference for open-source technologies due to budget constraints
- **License Restrictions**: Compliance with software license requirements and limitations
- **Version Compatibility**: Constraints on software version compatibility and updates
- **Integration Limitations**: Restricted integration with proprietary academic systems

### 2. Development Tools
- **Free Software**: Reliance on free development tools and IDEs
- **Limited IDE Features**: Constraints from using community edition development tools
- **Debugging Limitations**: Restricted access to advanced debugging and profiling tools
- **Collaboration Tools**: Limited access to premium collaboration and version control features

### 3. Database Constraints
- **Community Editions**: Use of community edition database management systems
- **Storage Limitations**: Database size and connection limitations
- **Performance Constraints**: Restricted database performance optimization capabilities
- **Backup Limitations**: Limited automated backup and recovery features

### 4. Machine Learning Constraints
- **Model Complexity**: Limited model complexity due to hardware constraints
- **Training Data**: Restricted access to large-scale training datasets
- **Computational Resources**: Limited GPU resources for deep learning model training
- **Framework Limitations**: Constraints on ML framework features and capabilities

## Data Constraints

### 1. Data Availability
- **Synthetic Data**: Reliance on synthetic or publicly available datasets for initial development
- **Limited Real Data**: Restricted access to real student data due to privacy concerns
- **Sample Size**: Limited sample size for model training and validation
- **Data Diversity**: Constraints on demographic and contextual data diversity

### 2. Data Quality Issues
- **Missing Values**: Incomplete data records affecting model accuracy
- **Noise and Errors**: Data quality issues requiring extensive preprocessing
- **Bias Concerns**: Potential biases in available training datasets
- **Label Availability**: Limited labeled data for supervised learning approaches

### 3. Privacy and Compliance
- **Data Anonymization**: Requirements for data anonymization affecting utility
- **Consent Limitations**: Constraints on data collection due to consent requirements
- **Retention Policies**: Data retention limitations affecting longitudinal analysis
- **Sharing Restrictions**: Inability to share data across research collaborations

## Financial Constraints

### 1. Budget Limitations
- **Zero Budget**: No financial allocation for software licenses or cloud services
- **Personal Funding**: Reliance on personal financial resources for any paid services
- **University Support**: Limited university funding for academic projects
- **Cost Optimization**: Need for cost-effective solutions throughout development

### 2. Resource Allocation
- **Free Services**: Dependence on free tiers of cloud services and tools
- **Trial Periods**: Limited access to premium software through trial periods
- **Educational Discounts**: Utilization of educational discounts where available
- **Open Source Alternatives**: Preference for open-source alternatives to commercial software

## Legal and Ethical Constraints

### 1. Institutional Requirements
- **Ethics Review**: Mandatory institutional ethics review and approval
- **Informed Consent**: Requirements for comprehensive informed consent procedures
- **Data Protection**: Compliance with institutional data protection policies
- **Research Integrity**: Adherence to academic research integrity standards

### 2. Regulatory Compliance
- **Privacy Laws**: Compliance with applicable privacy protection legislation
- **Data Residency**: Requirements for data storage within specific geographic boundaries
- **Age Restrictions**: Compliance with age-related data protection requirements
- **International Regulations**: Consideration of international data transfer regulations

## Technical Constraints

### 1. Performance Limitations
- **Response Time**: Target response times may be affected by hardware limitations
- **Concurrent Users**: Limited capacity for simultaneous user access
- **Data Processing**: Constraints on real-time data processing capabilities
- **Scalability**: Limited system scalability due to infrastructure constraints

### 2. Integration Challenges
- **API Limitations**: Restricted access to third-party system APIs
- **Format Compatibility**: Data format incompatibilities with external systems
- **Authentication Issues**: Challenges with user authentication across systems
- **Real-Time Synchronization**: Difficulties with real-time data synchronization

## Risk Mitigation Strategies

### 1. Technical Risks

- **Incremental Development**: Phased development to manage technical complexity
- **Prototype Testing**: Early prototyping to identify technical challenges
- **Documentation**: Comprehensive technical documentation for knowledge transfer

### 2. Project Risks
- **Time Management**: Strict project timeline management and milestone tracking
- **Scope Control**: Careful scope management to prevent feature creep
- **Quality Assurance**: Regular quality checks and testing procedures
- **Backup Planning**: Contingency plans for critical project components

### 3. Resource Risks
- **Alternative Tools**: Identification of alternative tools and technologies
- **Skill Development**: Plan for acquiring necessary technical skills
- **External Support**: Strategy for obtaining external technical assistance
- **Resource Optimization**: Efficient utilization of available resources

## Constraint Impact Assessment

### High Impact Constraints
1. **Academic Timeline**: Strict semester deadline affects all development activities
2. **Hardware Limitations**: Processing constraints affect model complexity and performance
3. **Data Availability**: Limited real data affects model accuracy and validation
4. **Budget Constraints**: Zero budget limits technology choices and infrastructure options

### Medium Impact Constraints
1. **Software Licenses**: License restrictions affect technology stack selection
2. **Institutional Approval**: Ethics review timeline affects project initiation
3. **Team Coordination**: Schedule coordination affects development efficiency
4. **Testing Resources**: Limited testing infrastructure affects quality assurance

### Low Impact Constraints
1. **Documentation Time**: Limited documentation time affects project completeness
2. **Presentation Preparation**: Viva preparation affects final development phase
3. **Tool Learning Curve**: New tool adoption affects development speed
4. **Network Limitations**: Bandwidth constraints affect system accessibility

- **Flexible Planning**: Adaptive project planning based on constraint evolution
