# UI Flow Overview: PRCI v2.0

## Overview

This document outlines the end-to-end interaction flow for PRCI v2.0's demonstration interface. The console-based UI provides a simple, academic-focused demonstration of the complete system workflow while maintaining appropriate boundaries and non-clinical framing.

## System Interaction Flow

### Complete Data Pipeline

**Input → Detection → Risk → Intervention → Personalization → Feedback**

The UI demonstrates the full PRCI v2.0 pipeline:

1. **User Input**: Academic situation description
2. **Detection Layer**: Text analysis for procrastination indicators
3. **Risk Assessment**: Risk level calculation based on detection results
4. **Intervention Engine**: Generation of supportive suggestions
5. **Personalization Engine**: Preference-based intervention ranking
6. **Feedback Collection**: User feedback for system improvement

### Step-by-Step Flow

**Step 1: User Input Collection**
- Console prompt for academic situation description
- Input validation and sanitization
- Clear instructions for appropriate input content
- Academic context framing for user guidance

**Step 2: Detection Processing**
- Text analysis through detection models
- Identification of procrastination indicators and root causes
- Structured detection result preparation
- Academic-focused interpretation of analysis results

**Step 3: Risk Assessment**
- Risk level calculation based on detection history
- Risk categorization (LOW/MEDIUM/HIGH)
- Risk factor identification and explanation
- Academic context consideration in risk evaluation

**Step 4: Intervention Generation**
- Rule-based intervention selection based on risk and causes
- Template-driven intervention message generation
- Safety filtering and ethical boundary enforcement
- Academic support framing of intervention content

**Step 5: Personalization Application**
- User preference consideration in intervention ranking
- Feedback-based intervention selection
- Personalized intervention presentation
- User control over personalization features

**Step 6: Feedback Collection**
- User feedback on intervention helpfulness
- Preference update based on user response
- System improvement through user input
- Transparent feedback processing and acknowledgment
