# Interaction Guidelines: PRCI v2.0

## Overview

This document outlines the interaction guidelines for PRCI v2.0's demonstration interface. The guidelines ensure appropriate academic support framing, maintain non-clinical boundaries, and provide consistent, supportive user interactions.

## Input Boundaries

### Content Guidelines

**Academic Focus Only**
- Input should relate to academic situations and challenges
- Focus on procrastination and productivity in educational contexts
- Avoidance of personal, medical, or clinical topics
- Academic goal-oriented input and discussion

**Appropriate Length and Detail**
- Concise descriptions of academic situations (50-200 words recommended)
- Sufficient detail for meaningful analysis and support
- Clear articulation of academic challenges and concerns
- Focused input relevant to procrastination and productivity

**Respectful Communication**
- Professional and respectful language and tone
- Avoidance of inappropriate or offensive content
- Constructive academic discussion and problem-solving
- Appropriate communication for educational support context

### Input Validation

**Content Filtering**
- Automatic filtering of inappropriate or harmful content
- Academic context verification and validation
- Safety checks for potentially concerning content
- Clear user guidance for appropriate input

**Format Requirements**
- Text-based input with clear structure and organization
- Readable formatting and presentation
- Appropriate language and terminology
- Consistent input format for system processing

## Language Tone and Communication Style

### Supportive Framing

**Encouraging and Positive**
- Supportive and encouraging language throughout interactions
- Positive reinforcement of user efforts and progress
- Constructive approach to academic challenges and solutions
- Empowering language that builds user confidence and agency

**Non-Judgmental Approach**
- Accepting and non-judgmental attitude toward user challenges
- Understanding and empathetic response to academic difficulties
- Respect for user experiences and perspectives
- Supportive environment for honest academic discussion

**Collaborative Orientation**
- Partnership approach to academic problem-solving
- Collaborative language that emphasizes working together
- User expertise and experience acknowledgment and validation
- Team-based approach to academic success and productivity

### Non-Directive Communication

**Suggestion-Based Language**
- Use of suggestive rather than directive language patterns
- "Consider trying..." instead of "You should..."
- "You might find it helpful to..." instead of "You must..."
- Respect for user choice and decision-making autonomy

**Option Presentation**
- Multiple options and approaches for academic challenges
- User choice in strategy selection and implementation
- Flexible suggestions that adapt to user preferences
- Respect for individual learning styles and approaches

**Question-Based Engagement**
- Open-ended questions to encourage user reflection
- Guided inquiry for self-discovery and insight
- User-driven problem-solving and solution development
- Facilitative approach to academic support

## No Emotional Labeling

### Academic Context Framing

**Behavioral Focus**
- Focus on observable behaviors and academic actions
- Avoidance of emotional labeling or psychological interpretation
- Academic performance and productivity emphasis
- Action-oriented discussion and problem-solving

**Situation-Based Discussion**
- Academic situation analysis and discussion
- Context-specific challenges and solutions
- Environmental and temporal factors consideration
- Practical, actionable academic support

**Productivity and Performance**
- Academic productivity and performance focus
- Task completion and time management discussion
- Study habits and strategies development
- Educational goal achievement and progress

### Professional Boundaries

**Non-Clinical Language**
- Avoidance of clinical or therapeutic terminology
- No psychological diagnosis or assessment language
- Professional academic support framing
- Educational and developmental language use

**Appropriate Expertise Recognition**
- Acknowledgment of academic support role and limitations
- Clear distinction from clinical or therapeutic services
- Professional boundary maintenance and communication
- Appropriate referral to professional services when needed

## No Urgency or Authority Framing

### Respectful Pace

**User-Controlled Interaction**
- User-paced interaction and progress
- No pressure or urgency in system responses
- Respect for user timing and availability
- Flexible interaction scheduling and pacing

**Gentle Encouragement**
- Supportive encouragement without pressure
- Motivational language that respects user autonomy
- Positive reinforcement without urgency
- Encouraging but non-demanding communication

**Flexible Timing**
- No time pressure or deadline emphasis
- User choice in interaction timing and frequency
- Flexible system availability and access
- Respect for user schedule and preferences

### Collaborative Authority

**Shared Decision-Making**
- Collaborative approach to academic strategy development
- User input and preference integration
- Shared responsibility for academic success
- Partnership-based problem-solving approach

**Expertise Acknowledgment**
- Recognition of user expertise in their own experience
- System expertise in academic support strategies
- Collaborative knowledge and experience sharing
- Mutual respect and learning orientation

## Mandatory Communication Elements

### Required Disclaimer Statement

**Non-Clinical Boundary Statement**
"The interface provides academic support suggestions and does not offer psychological or medical advice."

This statement must be included in:
- Initial system introduction and welcome message
- Intervention presentation and suggestion delivery
- System limitations and capabilities explanation
- User guidance and support resource information

### Academic Purpose Reinforcement

**Educational Focus Communication**
- Regular reinforcement of academic support purpose
- Clear communication of system educational objectives
- Emphasis on productivity and academic success goals
- Consistent academic context framing throughout interaction

## User Support and Guidance

### Clear Instructions

**Step-by-Step Guidance**
- Clear, step-by-step instructions for system interaction
- User guidance for each interaction phase and process
- Understandable directions for system navigation and use
- Consistent instruction format and presentation

**Help and Support Information**
- Clear help and support resource information
- User guidance for system features and capabilities
- Assistance options for technical or interaction issues
- Resource connection for additional academic support needs

### Error Handling and Recovery

**Graceful Error Management**
- Clear error messages and resolution guidance
- User-friendly error communication and explanation
- Step-by-step error resolution instructions
- Supportive error recovery and system continuation

**Input Correction Support**
- User guidance for input correction and improvement
- Clear feedback on input issues and requirements
- Constructive suggestions for input enhancement
- Supportive approach to input validation and correction

## Conclusion

The interaction guidelines ensure that PRCI v2.0's demonstration interface provides appropriate, supportive, and ethical academic support while maintaining clear boundaries and professional standards. The guidelines support consistent, high-quality user interactions that respect user autonomy and promote academic success.

The mandatory disclaimer statement and non-clinical framing ensure that the interface maintains appropriate boundaries while providing valuable academic support and productivity guidance within an ethical and professional framework.
